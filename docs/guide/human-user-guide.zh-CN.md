# 给普通用户的使用说明（最后更新：2026-04-16）

这套系统默认是给 Agent 来部署、运行和协作的。

如果你不会写代码，也没有关系。你的主要工作不是自己敲很多命令，而是把目标讲清楚，让 Agent 帮你完成。

## 你可以把它理解成什么

EasyAgentTeam 可以理解成一个“多位 AI 助手分工协作”的系统。

它通常会做这几件事：

- 帮你创建项目或工作区
- 帮你安排多个 Agent 分工
- 帮你执行项目任务或工作流
- 帮你记录过程，方便回头查看

先记住一句话：

`你负责讲清目标，Agent 负责部署、运行、检查和解释结果。`

## 最常见的使用方式

### 1. 让项目专用建项目 Agent 先帮你起步

你可以直接说：

```text
请先用项目专用建项目 Agent 帮我把项目工作区初始化好，并告诉我下一步该看哪里。
```

如果你想说得更明确一点，可以这样说：

```text
请帮我检查环境、安装依赖、启动系统，然后用建项目 Agent 初始化一个可用工作区，最后告诉我结果在哪里看。
```

### 2. 让 Agent 帮你介绍这个项目能做什么

```text
请用普通人能听懂的话介绍这个项目能做什么，重点说我能怎么用。
```

或者：

```text
请带我快速了解这个项目的主要功能，不要讲太多代码细节。
```

### 3. 让 Agent 帮你跑一轮标准验证

```text
请帮我跑一轮标准检查，并告诉我结果是成功还是失败。
```

## 运行后你主要看什么

你通常只需要看这几类结果：

- 是否启动成功
- 是否有报错
- 任务或流程是否完成
- 系统有没有留下结果记录

如果 Agent 说已经运行成功，你可以继续问：

```text
请告诉我现在去哪里看运行结果，用最简单的方式说明。
```

## 如果你想看界面，优先看哪里

普通用户先看下面几个位置就够了：

- `Projects`：看项目任务有没有推进
- `Workflow`：看流程有没有运行
- `task-tree`：看任务结构和完成状态
- `timeline`：看系统做过什么事
- `Settings`：看配置是否大体正常

如果你不知道该看哪个页面，可以直接问：

```text
我现在应该看哪个页面来确认系统是否正常？
```

## 遇到问题时，最简单的说法

你不需要先判断技术原因，直接把现象告诉 Agent 就可以。

例如：

```text
项目没有启动起来，帮我看看卡在哪里。
```

```text
流程没有继续往下跑，帮我检查现在阻塞在什么地方。
```

```text
页面能打开，但我不知道结果算不算正常，帮我判断一下。
```

```text
请不要改太多东西，先观察和分析问题原因。
```

## 推荐的提问方式

和 Agent 沟通时，尽量说清楚下面三点：

- 你想做什么
- 你现在看到什么现象
- 你希望 Agent 先做什么

一个好例子：

```text
我想确认这个项目能不能正常运行。现在页面已经打开，但我不知道系统是不是已经准备好。请先帮我检查运行状态，再告诉我下一步看哪里。
```

## 你不需要自己做的事

普通用户一般不需要自己做这些事：

- 不需要读大量代码
- 不需要自己判断复杂日志
- 不需要自己设计命令
- 不需要先懂项目架构再开始用

这些事情可以先交给 Agent。
